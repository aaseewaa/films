# Скрипты загрузки данных

Все скрипты запускаются из папки `backend/`, **с активным venv**, как Python-модули (`-m scripts.имя`). Это важно — иначе не сработают импорты `from app...`.

## load_tmdb.py — основной загрузчик

Тянет популярные фильмы из TMDB и пишет в БД через ORM. На двух языках (ru/en).

```bash
cd backend
source venv/bin/activate

# Пробный прогон: 1 страница = 20 фильмов, по 5 актёров
python -m scripts.load_tmdb --pages 1 --top-actors 5

# Боевой прогон: 15 страниц = 300 фильмов, по 10 актёров
python -m scripts.load_tmdb --pages 15 --top-actors 10
```

### Что делает

Для каждого фильма получает:
- название, описание, год, постер на ru и en
- режиссёров (всех)
- актёров (top N по billing order)
- жанры (с переводами)
- внешние id (tmdb, imdb)
- метаданные: рейтинг, бюджет, сборы, tagline

Записывает в БД:
- `entity` + `film` + `entity_translation` (×2 языка) для каждого фильма
- `entity` + `person` + `entity_translation` (×2 языка) для каждой новой персоны
- `film_person` — связи фильм-режиссёр и фильм-актёр
- `taxonomy_term` + `taxonomy_term_translation` — жанры
- `entity_taxonomy` — связи фильм-жанр

### Особенности

- **Идемпотентность.** Повторный запуск НЕ дублирует фильмы — проверяет `entity.external_ids->>'tmdb'`.
- **Кэш.** Все ответы TMDB кэшируются в `scripts/cache/tmdb/*.json`. Если что-то упало посередине — повторный запуск не будет дёргать API заново для уже скачанного.
- **По-фильмовая транзакция.** Если один фильм упал на ошибке — остальные продолжают грузиться.
- **Rate-limit.** Используется asyncio.Semaphore чтобы не превысить лимит TMDB.

### Сколько занимает

Примерно 1.5-2 секунды на фильм с холодным кэшем. То есть:
- 20 фильмов (1 страница) ≈ 1 минута
- 300 фильмов (15 страниц) ≈ 15-20 минут

### Если что-то пошло не так

```bash
# Очистить кэш и начать заново
rm -rf scripts/cache/tmdb

# Очистить БД (только в dev! на боевой не делай!)
psql -d FilmsDB_diplom -c "
TRUNCATE entity, app_user, app_role, language, taxonomy_term
RESTART IDENTITY CASCADE;
"
# потом снова прогнать seed.sql
```
